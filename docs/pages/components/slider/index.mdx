## Installation

Install the component from your command line:

```bash
npm install @interop-ui/react-slider
```

Then import the component in your project:

```js
import { Slider } from '@interop-ui/react-slider';
```

## Basic Usage

The most basic composition of a `Slider` component is as follows:

```jsx
// live
<Slider defaultValue={10}>
  <Slider.Track>
    <Slider.Range />
  </Slider.Track>
  <Slider.Thumb />
</Slider>
```

### Controlled state

The component will work as expected without any additional props or configuration, but if you need access to its state you can control it with the `value` and `onChange` props.

```jsx
// live
function ControlledSlider() {
  const [value, setValue] = React.useState(10);
  return (
    <Slider value={value} onChange={(newValue) => setValue(newValue)}>
      <Slider.Track>
        <Slider.Range />
      </Slider.Track>
      <Slider.Thumb />
    </Slider>
  );
}

// render
<ControlledSlider />;
```

## Styling

All `Slider` components are unstyled by default. In most cases, each component renders a single DOM node which can be styled however you'd style a JSX component. In the event that a `name` prop is provided to `Slider`, it will also render a visually hidden input for each thumb that will be used to pass values into a form.

### Basic style recommendations

`Slider` will require some styling to achieve basic functionality. The following recommendations serve as a good base to build on for your component.

```css
/* outer container styles */
[data-interop-ui-slider] {
  position: relative;
  display: flex;
  align-items: center;
  flex-shrink: 0;
  user-select: none;
  touch-action: none;
}

/* styles for horizontal sliders */
[data-interop-ui-slider][data-orientation='horizontal'] {
}

/* styles for vertical sliders */
[data-interop-ui-slider][data-orientation='vertical'] {
  flex-direction: column;
}

/* track styles */
[data-interop-ui-slider-track] {
  position: relative;
  flex-grow: 1;
}

/* range styles */
[data-interop-ui-slider-range] {
  position: absolute;
}

/* styles for horizontal slider range */
[data-interop-ui-slider-range][data-orientation='horizontal'] {
  height: 100%;
}

/* styles for vertical slider range */
[data-interop-ui-slider-range][data-orientation='vertical'] {
  width: 100%;
}

/* thumb styles */
[data-interop-ui-slider-thumb] {
  display: block;
}

/* Add recommended touch target size regardless of styled size */
[data-interop-ui-slider-thumb]::before {
  content: '';
  position: absolute;
  z-index: -1;
  width: 44px;
  height: 44px;
  top: 50%;
  left: 50%;
  transform: translate(-50%, -50%);
}
```

## Accessibility

- The user can touch or click anywhere on the `Slider` track to activate the thumb and update the value. Regardless of the visible size of the track or thumb, a sufficiently large touch target should exist to help users regardless of input modality. The touch target can be increased by adding invisible pseudo elements to the thumb and/or track to expand the boundaries of the `Slider`'s internal event handlers.
- The element with the accessible role of `"slider"` is actually the `SliderThumb`, not the `Slider` itself. This is because a `Slider` can have multiple thumbs that actually represent two independent values.
- The element rendered by `SliderThumb` sets a value for `aria-label` by default that represents the slider's range in a human readable format. This can be overridden by passing `aria-label` directly to `SliderThumb`.
- The element rendered by `SliderThumb` sets a value for `aria-valuemin` equal to the `min` prop passed to `Slider`.
- The element rendered by `SliderThumb` sets a value for `aria-valuemax` equal to the `max` prop passed to `Slider`.
- The element rendered by `SliderThumb` sets a value for `aria-valuenow` equal to the component's current value.
- The element rendered by `SliderThumb` sets a value for `aria-orientation` equal to the `orientation` prop passed to `Slider`.
- The `SliderThumb` must be in the document's tab sequence and has a `tabIndex` of `0` by default

### WAI-ARIA Guidance

`Slider` implements the [WAI-ARIA slider pattern](https://www.w3.org/TR/wai-aria-practices-1.2/#slider).
